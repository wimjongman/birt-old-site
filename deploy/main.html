Deploying BIRT

Overview

Integration Points

BIRT provides two main solutions for integrating with your application:

  • Using the BIRT Viewer web application, or a customized version
  • Using the BIRT Report Engine in your application.

Status

The deployment portions of the BIRT project are still being refined. The information here may (and probably will) change as Release 1.0 approaches. On the other hand, this means that there is still time to provide your comments and suggestions via the news group.

The BIRT Viewer

The BIRT viewer is a web application that ships with BIRT to perform report preview within Eclipse.

  • Deploying the BIRT Viewer explains how to deploy the BIRT viewer onto your own application server. It uses Apache Jakarta Tomcat as the sample deployment.
  • Understanding the Sample Viewer discusses the major components of the viewer application and how to customize them.
  • Viewer Pages discusses the JSP that make up the viewer. (Not yet available.)
  • The viewer code itself is available in the BIRT CVS repository in the org.eclipse.birt.report.viewer project. See the BIRT build instructions for information on how to access CVS and build the viewer.

The BIRT Viewer can be used in a variety of ways:

  • As a stand-alone, pre-built web application for running and viewing reports.
  • As a starter web application that you can customize to your needs.
  • As an example for learning how to build your own reporting web application, or to learn how to use the BIRT engine.
  • As a way to run a report using a URL. This option provides a simple way to integrate BIRT reporting into applications build using non-Java technology such as Perl, PHP or even static web pages.

The BIRT Engine SDK

You can also build an application that uses the BIRT Report Engine directly. This might be a web application, Eclipse plug-in or even a command-line application.

  • Installing the BIRT Engine SDK explains how to install the Engine SDK and how to use it within Eclipse. (Not yet available.)
  • Installing the BIRT Runtime explains how to install and configure the BIRT engine as part of your application. (Not yet available.)
  • Working with the BIRT Engine explains how to integrate the BIRT engine within your application. (Not yet available.)
  • The sample Engine application is a simple, command-line application that runs a report and illustrates the above topics. (Not yet available.)

The BIRT engine is simply a series of jar files and can be deployed within a wide range of Java applications:

  • Web application using your favorite application framework (JSP, servlet, JSF, Struts, etc.)
  • Traditional Java client application based on Swing.
  • Eclipse Rich Client Platform (RCP) application.
  • Even as a command-line utility.
