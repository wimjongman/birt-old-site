Integrating BIRT

Installing the BIRT Viewer in Tomcat

Using Tomcat

This page explains how to deploy the BIRT viewer to a J2EE container. We'll use Apache Tomcat, since it is open source and readily available. The same concepts, perhaps with different details, apply to other app servers. These instructions assume you'll install Tomcat on your own machine using the default port number of 8080.

The following instructions are for Tomcat version 5.5.7 but you should be able to use them for other versions with appropriate changes. If you don’t have Tomcat installed on your system you can download it from http://jakarta.apache.org/tomcat.

Prepare Tomcat

BIRT is tested for use with Java 1.4, but Tomcat 5.5 runs on Java 5 by default. Be sure to download and install the compatibility package from the Tomcat download page. Then, configure Tomcat to use the Java 1.4 runtime. The easiest way to do this is to use the Tomcat manager.

  • On Windows, use Start→All Programs→Apache Tomcat 5.5→Monitor Tomcat.
  • Stop Tomcat.
  • Switch to the Java page.
  • For Java Virtual Machine, enter the path to a Java 1.4 runtime.
  • Restart Tomcat.

Then, verify that the configuration is correct:

  • In your browser, open the main page for Tomcat. This is localhost:8080 by default.
  • Switch to the Status page. Make sure that the JVM version is from the 1.4 series.

Install the Viewer

The next step is to deploy the BIRT Viewer application.

Release 1.0M1 Note: The download page does not include a viewer build. Instead, we'll use the viewer packaged with the Report Designer. These instructions will change once a viewer build is available.

  • Locate the Eclipse plugins directory.
  • Find the org.eclipse.birt.report.viewer_0.0.1 plugin. Open this directory.
  • Copy the birt-viewer directory to the webapps directory of your Tomcat installation.
  • Stop, then restart Tomcat.
  • Display the Tomcat manager application and verify that the Viewer is deployed: http://localhost:8080/manager/html.
  • Verify that birt-viewer is listed as an application, and that it is running.

If you choose to put the Viewer into some other location, you'll need to use a context entry within the server.xml file to indicate the deployment location. See Tomcat documentation for details.

Install your JDBC Drivers

Follow the JDBC driver install process to add your JDBC drivers to the Viewer. The drivers go in the WEB-INF/plugins/drivers directory. (Note: you must install the drivers both into the Eclipse-based design environment and the deployed viewer environment; these two environments do not share files.

Test the Viewer

We'll test the viewer by creating a test report and deploying it into the Viewer directory.

  • Download the test report design into your c:\temp directory.
  • If you've installed everything in its default location, then click on the following link. If you've changed anything, then copy the following URL into your browser and make the needed changes.
    http://localhost:8080/birt-viewer/run?__report=C%3A%5Ctemp%5Ctest.rptdesign

Build S20050328 Note: The viewer currently requires an absolute path to the report on the server machine. Upcoming versions will support relative paths.

Build S20050328 Note: The above link will run the report with default parameter values. The link to display the parameter page is not yet available in this build.

The report should run and appear in your browser.
