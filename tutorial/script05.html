Building a Data Set

Build a Data Set

Now we're ready to build and test our data set. This is a two-step process. First, we build the bulk of an SQL query using a wizard, then we fine tune it and test it using the data set editor. Let's build the data set:

  • Find or open the Data Explorer view.
  • Select the Data Sets node in the tree view.
  • Right-click to open the context menu. (We'll just say "open the context menu" or "choose x from the context menu" from now on.)
  • Select New Data Set. The New Data Set dialog will appear.
  • Enter "Customers" for the Data Set Name field.
  • Ensure that the Data Source field shows the sfdata data source created above.
  • Ensure that the Data Set Type field shows "JDBC select data set". This builds a data set for an SQL SELECT query.
  • Click Next.

Choose Tables

The next page lets you select one or more tables for your query. For our first report, we'll select just one table: Customers.

  • Choose "Tables" within the Select Type field.
  • Type "%" (without the quotes) into the Apply Filter field.
  • Click Apply Filter to find all tables.
  • Select the customers table.
  • Click the > button to add it to the query.
  • Click Next.

Choose Columns

You can choose any number of columns to appear in your report. Normally, you would choose only those columns that are actually needed by the report. For our first report, we'll choose all columns.

  • Scroll the list of columns up so you can see the customers table entry.
  • Select the customers table entry.
  • Click the > button to add all customer columns to your query.
  • Click Finish.
