Building a Data Source

About Data Sources and Data Sets

BIRT reports use a data source to connect to a database or other data provider. BIRT uses a data set to retrieve data from the data provider. Think of the data source as a connection, and the data set as a query.

Build a Data Source

Let's build our data source:
  • Open the Data Explorer, if it is not already visible.
  • Select the Data Sources node within the tree view.
  • Right-click to open the context menu.
  • Choose New Data Source.
  • The New Data Source dialog appears.
  • Choose JDBC Data Source from the list.

    Release 1.0M1 Note: Only JDBC is supported at present. BIRT's Open Data Access (ODA) framework allows developers to create other data source types.

  • Click Next.
  • Enter "sfdata" for the Data Source Name field.
  • Choose "sun.jdbc.odbc.JdbcOdbcDriver" in the Driver Class field.
  • Type "jdbc:odbc:sfdata" in the URL field.
  • Type "admin" in the User Name field.
  • Leave the Password field blank.
  • Click Test Connection to test your connection. (If it fails, double-check the above information, or see the instructions for setting up sfdata.)
  • Click Finish

Editing your Data Source

Notice that your new data source appears in the Data Explorer.
  • Find or open the Data Explorer.
  • Open the Data Sources node
  • Double-click the sfdata data source.
  • The Data Source editor opens. You can make any needed changes. For now, there is nothing to change.
  • Click OK to save your changes.
