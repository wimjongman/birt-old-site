OdaDataSource Element

Description

The ODA Data Source element represents an Open Data Access (ODA) connection.  It is described by a driver name and optional sets of driver-specific properties.

An ODA driver may statically define driver-specific connection properties by implementing the ODA model extension point (org.eclipse.birt.report.model.odaDriverModel). See the BIRT Extensions functional specification document for more detail description.  By implementing this extension point, an ODA driver’s extension plug-in manifest (plugin.xml) could statically define driver-specific connection properties. BIRT supports these driver-defined properties as ROM properties with all their built-in features, such as direct editing from the BIRT designer’s property sheet. BIRT uniquely identifies each of these extension-specified properties within the scope of its ODA extension. From the UI perspective, for ease of identifying these driver-specific connection properties, one could define them within a group, and name it something like "connectionProperties". At Factory time, each of these static driver-defined properties, with its defined property name and corresponding value, is passed to the ODA run-time driver as runt-time properties to open a data source.

The following plug-in manifest segment demonstrates a simple example of such static property definitions.

<extension
      name="BIRT ODA-JDBC Bridge"
      point="org.eclipse.birt.report.model.odaDriverModel">
  <odaDataSource
        extensionName="jdbc">
    <propertyGroup
          name="connectionProperties">
      <property
            name="ODA:driver-class"
            type="string"
            defaultValue="org.apache.derby.jdbc.EmbeddedDriver"/>
      <property
            name="ODA:url"
            type="string"
            defaultValue="jdbc:derby:net://localhost:1527/db"/>
       <property
            name="ODA:data-source"
            type="string"/>
       <property
            name="ODA:user"
            type="string"/>
       <property
            name="ODA:password"
            type="string"/>
     </propertyGroup>
  </odaDataSource>
</extension>

Description

A list of dynamic name-value structures (ExtendedProperty structures) that specify private runtime data or attributes specific to the ODA connection. This is an optional element that an ODA driver may use to store and pass pertinent driver-specific attributes from its designer tool to the runtime driver. The content of this element is specific and private to an ODA driver, whose designer tool can dynamically define and update them as appropriate. These name-value pairs are stored in the report design file as a persistent service, and are passed to the ODA runtime driver at Factory time as connection properties.
