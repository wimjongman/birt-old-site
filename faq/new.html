Getting Started

Q: How should I get started with BIRT?

Many people want to try out BIRT with a simple report design, then see that report work on their app server before diving into the details of BIRT. Here are some suggested steps for getting started. First, try out the BIRT Designer:

  • Install Eclipse, BIRT and its prerequisites.
  • Locate and install the JDBC driver for your database. Install them in the BIRT designer (same page).
  • Walk through the BIRT tutorial to get an overview of the designer.

Next, try out the BIRT viewer on your app server:

  • Install Tomcat & the BIRT viewer.
  • Install & try out the sample report. (same page as above.)
  • Install your JDBC drivers into the Viewer. (same page as above.)

Now you're ready to try out a report specific to your application:

  • Create a simple report, perhaps a listing of some table, using the BIRT designer.
  • Preview the data to ensure the connection stuff works.
  • Preview the report itself to verify your layout.
  • Create a "reports" directory within the birt-viewer web app created above.
  • Copy your "first report" to this new reports folder.
  • Run your report using Tomcat. (Use the URL as on the Viewer setup page, but use reports/yourReport.rptdesign as the report name, where yourReport.rptdesign is the name of your report.)

At this point, you're ready to work on greater integration:

  • Learn about BIRT Viewer URLs (for passing report parameters).
  • Some brief hints for using the Viewer with PHP, if that is your server environment.
